# Parent Mobile Application Specification

## Overview

The Parent Mobile App provides guardians with real-time visibility into their child's school transportation. It shows bus location, estimated arrival times, delay alerts, and ride history. The design prioritizes trust, clarity, and ease of use for busy parents.

---

## User Stories

### Daily Workflow
- As a parent, I want to see all my children's transportation information in one place
- As a parent, I want to know when the bus is approaching so I can send my child out on time
- As a parent, I want to be notified immediately if the bus is delayed
- As a parent, I want confirmation that my child was picked up and dropped off safely
- As a parent, I want to view past rides to verify service quality
- As a parent, I want to update my notification preferences

### On-Demand Requests
- As a parent, I want to request special transportation for my child
- As a parent, I want to track the status of my request

## Screens

### 1. Home Screen

**Purpose:** Overview of all linked children and their transportation status.

**Layout:**
- Header: App logo, notification bell with unread count
- Child cards: One card per linked child
  - Child photo (if uploaded) or initials avatar
  - Name and grade
  - Current status: "On Bus", "At School", "Heading Home", "No Active Run"
  - Next event: "Pickup at 7:27 AM" or "Dropoff at 3:45 PM"
  - Tap to view detailed tracking
- Quick actions: "Request Ride", "View History", "Contact Dispatch"
- Empty state: "No children linked. Add your child to get started."

---

### 2. Child Linking Screen

**Purpose:** Securely link a parent's account to their child.

**Layout:**
- Instructions: "Enter your child's student ID and the verification code provided by the school or transportation office."
- Form fields:
  - Student ID
  - Verification code (6 digits)
  - Relationship to child: Mother, Father, Guardian, Grandparent, Other
- "Link Child" button
- Success state: "Emma Johnson has been linked to your account."

**Security:**
- Verification codes are generated by the school administrator or dispatcher
- Codes expire after 7 days
- Each child can have up to 4 linked guardians
- Primary guardian can approve additional links

---

### 3. Tracking Screen

**Purpose:** Real-time map showing the bus location and route progress.

**Layout:**
- Map: Google Maps with bus icon, route polyline, stop markers
- Info panel (bottom):
  - Child name and current status
  - Run ID and vehicle number
  - Driver name
  - Next stop name and ETA
  - Stops remaining count
- Refresh button: Manual refresh for latest data
- "View Route Details": Opens timeline view

**Interactions:**
- Map auto-centers on bus location
- Bus icon moves as GPS updates arrive
- Tapping a stop marker shows stop name and scheduled time
- Pull-to-refresh updates data

---

### 4. Route Timeline Screen

**Purpose:** Show the planned route with actual progress.

**Layout:**
- Vertical timeline of all stops
- Each stop: Time, name, type, status
- Status indicators:
  - Completed: Checkmark with actual time
  - Current: Highlighted with "In Progress"
  - Upcoming: Gray with scheduled time
- Child's stop: Highlighted with special indicator

---

### 5. Notifications Screen

**Purpose:** History of all alerts and messages.

**Layout:**
- Filter tabs: All, Delays, Pickups, Dropoffs, General
- Notification list: Date, type, message, read/unread status
- Tap to expand full details
- Swipe to dismiss
- "Mark All Read" button

---

### 6. History Screen

**Purpose:** Review past rides for all linked children.

**Layout:**
- Date filter: Today, This Week, This Month, Custom Range
- Ride list: Date, run ID, pickup time, dropoff time, driver name
- Tap to view full ride details including stop timeline
- Export option: PDF ride log for records

---

### 7. On-Demand Request Screen

**Purpose:** Request special transportation.

**Layout:**
- Child selector: Which child needs transportation
- Trip details:
  - Pickup address (with map pin selection)
  - Dropoff address (with map pin selection)
  - Date needed
  - Preferred time
  - Passenger count
  - Wheelchair needed (toggle)
  - Purpose (field trip, medical appointment, etc.)
- "Submit Request" button
- Confirmation: Request number, estimated response time
- Status tracking: Pending, Approved, Assigned, Completed

---

### 8. Profile Screen

**Purpose:** Manage account and preferences.

**Layout:**
- Profile info: Name, phone, email
- Linked children: List with option to unlink
- Notification preferences:
  - Push notifications: On/Off
  - SMS notifications: On/Off
  - Email notifications: On/Off
  - Delay alerts: On/Off
  - Pickup confirmations: On/Off
  - Dropoff confirmations: On/Off
- Language preference (future)
- Help and support
- Logout

## Key Interactions

### Receiving a Delay Alert
1. System detects delay (driver report or GPS deviation)
2. Push notification arrives: "Route E-613 is running 15 minutes late due to traffic. New ETA: 8:00 AM."
3. Parent taps notification
4. App opens to tracking screen showing updated ETA
5. Parent can view delay reason and expected resolution

### Real-Time Bus Approach
1. Bus is 5 minutes from child's stop
2. Push notification: "Bus E-613 is approaching Stop A. Please have Emma ready."
3. Parent sees live map with bus icon approaching
4. Parent sends child out
5. Confirmation arrives: "Emma Johnson was picked up at 7:27 AM."

### Requesting a Ride
1. Parent taps "Request Ride" from home screen
2. Fills in pickup, dropoff, date, time
3. Submits request
4. Confirmation screen shows request number (e.g., ODR-2026-0042)
5. Dispatcher reviews and approves
6. Parent receives notification: "Your ride request has been approved. Driver: John Smith. Vehicle: V-104."

---

## Notification Types

| Type | Trigger | Channels | Priority |
|------|---------|----------|----------|
| Bus Approaching | 5 minutes from stop | Push | High |
| Delay Alert | Run delayed 5+ minutes | Push, SMS | High |
| Pickup Complete | Driver marks stop complete | Push | Medium |
| Dropoff Complete | Driver marks stop complete | Push | Medium |
| Run Started | Driver begins route | Push | Low |
| On-Demand Confirmed | Dispatcher approves | Push, SMS | High |
| On-Demand Denied | Dispatcher rejects | Push, SMS | Medium |

---

## Privacy and Security

- Parents can only view children they have verified via linking code
- Location data is only shared while the bus is on an active run
- Historical location data is purged after 90 days
- Parents cannot see other children's information
- Account deletion removes all personal data per policy
